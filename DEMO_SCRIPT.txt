[0:00 — HOOK]
Every day, thousands of researchers and students get overwhelmed trying to parse complex scientific publications. The Scientific Paper Analyzer solves this by automatically extracting methodology, equations, and results from PDFs, summarizing them under a strict human-in-the-loop validation pipeline.

[0:20 — WHAT IT IS]
This project is built using the Google Agent Development Kit, or ADK, for the "Agents for Good" track. It leverages a multi-agent workflow architecture to parse papers securely, let the researcher review and iterate on summaries, and save the final structured reports locally.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's trace how a request moves through the system.
First, the user submits a prompt, which lands at our security checkpoint. This node scrubs personal identifiers like emails and ORCID IDs, guards against prompt injections, and applies an academic content filter to reject off-domain uploads.
Once cleared, the input moves to the main orchestrator agent.
The orchestrator coordinates the parsing and summarizing stages by invoking two specialized sub-agents via AgentTool: first, the pdf_analyzer, which interacts with our local MCP server to read and extract text from the PDF, and second, the summarizer, which constructs a highly structured 5-section summary.
Finally, the flow halts at the human review node for approval, before routing to the report_writer, which uses our MCP write_report tool to save a clean markdown document.

Here is where the magic happens: the workflow pauses at a human review step using RequestInput. As a researcher, I don't have to accept raw model output. If the summary misses a crucial constraint, I can reject it and supply feedback, routing the summary back to the summarizer for revision.
